整数的二进制表示方式

  • 整数的二进制表示方式
    一般用于表示带符号整数,通常有原码，反码，补码三种表示方式，计算机选择补码方式表示。
编码方式 编码方式 最大值 最小值 0
原码 1/0位二进制 01111111:127 11111111:-127 00000000
10000000
反码 负数:原码取反
正数:原码		 01111111:127 10000000:-127 00000000
11111111
补码 负数:原码取反+1
正数:原码		 01111111:127 后面讲 00000000
00000000
  • 原码与补码存在的问题：
    • 原码：无法计算负数 比如 (原)10000001(-1) + (原)10000010(-2) = (原)00000011(3)
    • 反码：无法计算正数加负数 比如 (反)11111110(-1) + (反)00000010(2) = (反)00000000(0)
  • 补码的最小值为10000000:-128可以理解为一种特殊性，原因简单分析一下:
    • 三种编码方案都采用首位作为符号位，所以剩下的位数共有(2**7)2=256种组合，也就是模为256，理论上可以标示256的数字。
    • 原码和反码(-127~+127)范围只有255个数字，因为0占掉了2个位置。
    • 补码0只占一个位置,所以选用一个极为特殊的(补)10000000作为-128。 这个补码很特殊，他们原码和补码是一样。
  • 总结下补码的特性：
    • 补码的补码等于源码
    • 补码0只有一种表示形式
    • 补码的范围与数学意义上的模相同。

整数类型的运算

  • 加减法不说，这两个运算比较简单。
  • 乘法与除法的基本原则：把除法转成乘法，乘法转成加法，减法也转成加法。
  • 位运算：直接对整型的二进制位进行操作。
    • 与（&）：都为1则为1，否则0
      • 清零，或者部分位清零，比如：10101010 & 00001111 = 00001010，此为前4位清零
    • 或 (|）：只要有一个为1则为1，否则0
      • 置一，或者部分位置一，比如：10101010 | 00001111 = 10101111，此为后4位置一
    • 异或（^）：相同为0，不同为1
      • 取得原值，与0异或得原址，比如：10101010 ^ 00000000 = 1010101
      • 取反，与1异或取反，比如：10101010 ^ 11111111 = 01010101
    • 非（～）：取反
  • 位移操作：
    • 左移（<<）：正数负数采用同样的方式，左移指定位数(超出范围取模)，高位抛弃，低位补0
      • 比如：00000101(5) << 1 = 00001010(10);00000101(5) << 2 = 00010100(20)
      • 如果字长为8位，则 << 1 与 << 9 相同。
      • 左移是乘2，公式 y = x * (2**位数），左移1位相当于乘2，左移2位相当于乘4
    • 带符号右移（>>）：右移动指定位数，高位补符号位，低位抛弃
      • 比如：00000111(7) >> 1 = 00000011(3); 00000111(7) >> 2 = 00000001(1)
      • 右移是除2取商，公式 y = 商(x / (2**位数))
    • 无符号右移（>>>）：右移动指定位数，高位补0，低位抛弃
  • 乘法操作-累加被乘数：
    • 上一部分内容我们知道左移动相当于乘2,也相当于与自己相加。但是无法进行*3操作
    • 假设我们要处理x7,则x7 = x ((22) + (21) + (2**0)) = x(22) + x*(21) + x*(2**0) = x<<2 + x<<1 + x<<0
  • 除法操作-累减除数直到结果<被除数：
